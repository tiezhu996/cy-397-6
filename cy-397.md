## 项目主要需求

### 项目概述
**项目类型：纯后端服务**
搭建一个合同模板生成与法律工单 API 服务，提供常用合同模板管理、合同填充生成、法律咨询工单提交与跟踪的后端接口。

### 功能需求

1. **合同模板管理**：管理合同模板库（租赁合同/劳动合同/借款合同/合作协议/保密协议），模板包含占位变量（甲方/乙方/金额/日期等）。
2. **合同生成**：用户选择模板并填入变量值，系统生成完整合同文本（纯文本/HTML 格式），支持导出 PDF（使用 wkhtmltopdf）。
3. **合同签署状态管理**：合同状态流转（草稿→待签署→已签署→已过期），记录签署时间和签署方信息。
4. **法律工单系统**：用户提交法律咨询工单（问题类型：劳动纠纷/合同纠纷/房产纠纷/知识产权/其他 + 问题描述 + 附件），分配工单状态。
5. **工单流转**：工单状态管理（待处理→处理中→已回复→已关闭），支持添加回复记录（文字 + 附件）。
6. **常见法律知识库**：管理法律FAQ 条目（分类 + 问题 + 解答），支持关键词搜索。
7. **用户合同库**：用户查看自己创建的所有合同列表，按状态筛选，支持合同模板收藏。

### 技术要求

- 后端框架：Spring Boot + Java 17
- ORM：MyBatis-Plus
- 数据库：MySQL 8.0
- 认证：JWT
- PDF 生成：wkhtmltopdf（Docker 容器内安装）
- 代码规范：严格遵循单一职责原则，每个文件只负责一个功能；常量（错误码、合同状态枚举、工单类型）统一维护在 `backend/src/main/java/*/constants/` 目录下；日志统一使用 SLF4J + Logback，配置在 `backend/src/main/resources/logback-spring.xml`；异常信息统一定义在 `backend/src/main/java/*/exception/`，通过 @ControllerAdvice 全局异常处理器统一捕获并返回标准格式响应。

## 部署要求

项目必须支持 Docker Compose 一键部署。

需要提供以下文件：
- 根目录 `docker-compose.yml`（编排后端 + 数据库）
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=contractapi`
- `backend/Dockerfile`（需安装 wkhtmltopdf）
- `.env.example`

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: contractapi`
- 容器名带 `contractapi-` 前缀
- 数据库数据使用命名卷持久化
- 数据库配置 `healthcheck`
- 端口映射：
  - 后端：19412:8080
